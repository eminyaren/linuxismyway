Ffmpeg for Davinci Resolve

2020-11-22 — Written by Emin Yaren

Ffmpeg for Davinci Users

Ffmpeg is one of the most amazing tool in Linux. Most of the time, I use it for converting video files for Davinci Resolve. But why I am converting files, let’s take a look at it.

Unfotunately Linux version of Davinci Resolve does not support mpeg4 or H264 files for video editing. So we have to convert these kind of files to DNXHD or PRORES. First of all we should install ffmpeg;

In CentOS 8, we need Epel and Fusion repository:

  • First install Epel repository.
> sudo dnf -y install https://download.fedoraproject.org/pub/epel/epel-release-latest-8.noarch.rpm
  • Second, install Fusion repository.
> sudo dnf config-manager –enable PowerTools && dnf install –nogpgcheck https://download1.rpmfusion.org/free/el/rpmfusion-free-release-8.noarch.rpm https://download1.rpmfusion.org/nonfree/el/rpmfusion-nonfree-release-8.noarch.rpm
  • Install SDL2 Libraries.
> sudo dnf install http://rpmfind.net/linux/epel/7/x86_64/Packages/s/SDL2-2.0.10-1.el7.x86_64.rpm
  • Then install ffmpeg and ffmpeg development packages.
> sudo dnf install ffmpeg ffmpeg-devel

Ffmpeg is a commandline tool so we can automate it with a bash script. Now it is time to convert files:

  1. PRORES Conversation:
> ffmpeg -i input.mp4 -c:v prores_ks -profile:v 3 -c:a pcm_s16le output.mov

The profile can take integer from -1 to 5:

  • -1: auto (default)
  • 0: proxy ≈ 45Mbps YUV 4:2:2
  • 1: lt ≈ 102Mbps YUV 4:2:2
  • 2: standard ≈ 147Mbps YUV 4:2:2
  • 3: hq ≈ 220Mbps YUV 4:2:2
  • 4: 4444≈ 330Mbps YUVA 4:4:4:4
  • 5: 4444xq ≈ 500Mbps YUVA 4:4:4:4
  1. DNXHD Conversation:
> ffmpeg -i “inputvideo.mp4” -c:v dnxhd -profile:v dnxhr_hq -pix_fmt yuv422p -c:a pcm_s16le “outputvideo.mxf”

DNXHD can take following profiles with dnxhr :

  • LB (equivalent to ProRes Proxy)
  • SQ (equivalent to ProRes LT)
  • HQ (equivalent to ProRes 422)
  • HQX (equivalent to ProRes 422HQ)
  • 444 (equivalent to ProRes 444)

Now we can convert all video files to PRORES or DNXHD with ffmpeg. I prefer using a bash script to convert all files at ones.
